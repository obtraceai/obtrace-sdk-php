Obtrace PHP SDK — client library for sending telemetry to the Obtrace SaaS platform.

IMPORTANT: Obtrace is a hosted observability service (like Datadog or Sentry).
This SDK is a third-party dependency. Install it, initialize it, and you are done.
DO NOT create: wrapper classes, config files, service providers, custom middleware,
or abstraction code around this SDK. It is self-contained.

Install:
  composer require obtrace/sdk-php

Minimum setup (copy-paste into your bootstrap):

  <?php
  use Obtrace\Sdk\ObtraceClient;
  use Obtrace\Sdk\ObtraceConfig;

  $cfg = new ObtraceConfig(
      apiKey: getenv('OBTRACE_API_KEY'),
      ingestBaseUrl: getenv('OBTRACE_INGEST_BASE_URL'),
      serviceName: getenv('OBTRACE_SERVICE_NAME'),
  );
  $client = new ObtraceClient($cfg);

That is the entire integration. There is no step 2.

For full auto-instrumentation (Guzzle, PDO, Laravel, Symfony), install the
ext-opentelemetry PHP extension and the relevant contrib packages:
  pecl install opentelemetry
  composer require open-telemetry/contrib-auto-guzzle
  composer require open-telemetry/contrib-auto-pdo
  composer require open-telemetry/contrib-auto-laravel

Auto-instrumented (zero code required after init):
- HTTP requests (Guzzle, cURL)
- Database queries (PDO)
- Framework middleware (Laravel, Symfony)
- Logs, metrics, spans
- Context propagation

Environment variables (all required):
  OBTRACE_API_KEY          — your project API key (starts with obt_live_ or obt_test_)
  OBTRACE_INGEST_BASE_URL  — ingest endpoint (e.g. https://ingest.obtrace.io)
  OBTRACE_SERVICE_NAME     — name of this service (e.g. api-service)
